## Bugs Found & Fixed

### BUG-001 · `geo full` — `AttributeError: 'AuditResult' object has no attribute 'vertical'`

**Severity**: High — crashes `--output` entirely; silently drops the header in rich mode  
**Commands affected**: `geo full` (all invocations)

**Root cause**  
`fullaudit_cmd.py` referenced `result.vertical` in two places, but `AuditResult`
has no `vertical` field. The correct field is `result.vertical_profile.vertical`
(a nested `VerticalAuditResult` dataclass added in v4.10).

Occurrences:
- `_print_rich` header panel — line ~253
- `_build_markdown` — line ~1033 (`--output` path)

The header access was inside a `try/except` (added in the same session), so it
emitted a stderr warning but continued rendering. The markdown path was unguarded
and caused a hard `AttributeError`, meaning `--output` always crashed.

**Fix applied** — `src/geo_optimizer/cli/fullaudit_cmd.py`

```python
# Before (both locations)
result.vertical or 'auto'

# After — header panel
_vert = getattr(result.vertical_profile, "vertical", None) or "auto"
f"  •  {result.page_size:,} bytes  •  {_vert} vertical"

# After — _build_markdown
f"**Vertical**: {getattr(result.vertical_profile, 'vertical', None) or 'auto'}"
```

### BUG-002 · `geo full --vertical` — invalid values silently fall back to `generic`

**Severity**: Medium — silent data quality issue; user gets wrong vertical without warning  
**Commands affected**: `geo full --vertical <value>`

**Root cause**  
The `--vertical` option in `geo full` was declared as plain `TEXT` with no
`click.Choice` constraint. Passing `--vertical saas` (instead of the valid
`saas-technology`) went through to `audit_vertical_profile`, which checked it
against `SUPPORTED_VERTICALS`, found no match, and silently returned `generic`.
The user saw `generic vertical` in the header with no indication their input
was invalid.

Every other command that accepts `--vertical` (`geo apply`, `geo autopilot`,
`geo rivalry`) used `click.Choice` — `geo full` was the only exception.

**Fix applied** — `src/geo_optimizer/cli/fullaudit_cmd.py`

```python
# Before
@click.option(
    "--vertical",
    default="auto", show_default=True,
    help="Vertical override (e.g. real-estate-proptech, saas, e-commerce).",
)

# After
@click.option(
    "--vertical",
    default="auto", show_default=True,
    type=click.Choice([
        "auto", "generic",
        "ecommerce-retail", "travel-hospitality", "healthcare-dental",
        "real-estate-proptech", "legal-professional-services",
        "manufacturing-industrial-b2b", "financial-services-insurance",
        "saas-technology", "education-edtech-k12", "local-home-services",
    ], case_sensitive=False),
    help="Vertical override.",
)
```

Now invalid values produce a clear Click error immediately:
```
Error: Invalid value for '--vertical': 'saas' is not one of 'auto', 'generic', ...
```

### BUG-003 · `_print_rich` had no per-section error guards (pre-session)

**Severity**: High — any exception inside `_print_rich` crashed the entire rich output;
user saw nothing instead of a partial report  
**Commands affected**: `geo full`

**Root cause**  
`_print_rich` rendered everything into a `StringIO` buffer and flushed it at the
very end. If any section raised (e.g. BUG-001 in the header), the flush never
happened and nothing was displayed. There was an outer `try/except` in `full()`
that fell back to `_print_plain`, but `_print_plain` was itself incomplete
(missing per-category bar details visible in `geo audit`).

**Fix applied** — same session, before E2E test run  
Each major section in `_print_rich` now has its own `try/except` + `_flush()` call:

```python
def _flush():
    chunk = buf.getvalue()
    if chunk:
        click.echo(chunk, nl=False)
    buf.truncate(0)
    buf.seek(0)

def _section_error(label: str, exc: Exception) -> None:
    click.echo(f"  [render error in {label}: {exc}]", err=True)

# Each section:
try:
    con.print(...)
except Exception as exc:
    _section_error("section name", exc)
_flush()
```

Sections guarded: header, score overview, rival comparison, score breakdown,
GEO card, copywriting card, content strategy card, AI presence card,
CRO/perf card, priority actions, footer.

### BUG-004 · `_print_plain` used as fallback but missing full GEO category breakdown (pre-session)

**Severity**: Medium — fallback output was inferior to `geo audit` plain output  
**Commands affected**: `geo full` (fallback path only)

**Root cause**  
`_print_plain` in `fullaudit_cmd.py` had a table-style GEO breakdown but was
missing the per-check detail lines that `geo audit`'s plain formatter shows
(individual robots.txt bot names, llms.txt sub-checks, etc.). Using `_print_plain`
as a fallback from the rich renderer was therefore producing an inferior and
confusing output — it looked like a different, less detailed tool.

**Fix applied**  
Removed `_print_plain` from the `full()` command call path entirely. The
`--format plain` option was also removed. `_print_rich` now handles its own
errors section-by-section (BUG-003 fix), so a rich fallback is no longer needed.

## Infrastructure Note

`geo full` and `geo marketing` are new commands (untracked in git at test time).
`uv tool install --force .` excludes untracked files from the sdist build, so
the installed `geo` binary didn't include these modules until they were copied
manually to the site-packages directory. Permanent fix: stage/commit these files
so they're included in the next `uv tool install`.
